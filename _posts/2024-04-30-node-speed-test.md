## 为什么要进行速度测试？

节点速度直接影响您的网络体验。通过测速可以：
- 🚀 找到最快的节点
- 📊 监控节点性能变化
- 🔧 优化网络配置
- 💡 选择最佳的机场服务

## 测速工具介绍

### 1. Clash 内置测速

**优点**：
- 集成在应用内
- 操作简单
- 实时显示结果

**使用方法**：
```
Clash → 代理 → 长按节点 → 测试延迟
```

- [Speedtest](https://www.speedtest.net/)
- [Fast.com](https://fast.com/)

```bash

# 使用 curl 测试下载速度
curl -w "@curl-format.txt" -o /dev/null -s https://www.google.com

```

## 测速指标解读

### 1. 延迟（Latency）

**定义**：数据包从源到目的地的往返时间

| 延迟范围 | 体验 | 适用场景 |
|---------|------|---------|
| < 50ms | 极佳 | 实时游戏、视频通话 |
| 50-100ms | 很好 | 一般浏览、流媒体 |
| 100-200ms | 良好 | 下载、邮件 |
| > 200ms | 较差 | 不推荐使用 |

### 2. 下载速度（Download）

```
推荐速度：
- 网页浏览：1-5 Mbps
- 标清视频：2-5 Mbps
- 高清视频：5-10 Mbps
- 4K 视频：25+ Mbps
```

### 3. 上传速度（Upload）

```
推荐速度：
- 邮件、文件上传：1-2 Mbps
- 视频通话：2-5 Mbps
- 直播：5-10 Mbps
```

### 4. 丢包率（Packet Loss）

```
理想情况：0% 丢包
可接受：< 1% 丢包
不可接受：> 5% 丢包
```

## 完整的测速流程

### 步骤 1：准备工作
```
1. 关闭其他应用
2. 连接稳定的 WiFi 或有线网络
3. 清空浏览器缓存
4. 关闭下载任务
```

### 步骤 2：选择节点
```
1. 列出所有可用节点
2. 记录节点信息
3. 逐个进行测试
```

### 步骤 3：进行测速
```
1. 连接到节点
2. 等待 30 秒稳定连接
3. 打开测速网站
4. 开始测速
5. 记录结果
```

### 步骤 4：数据分析
```
1. 比较各节点速度
2. 计算平均值
3. 选择最佳节点
4. 定期重复测试
```

## 速度优化建议

### 1. 选择合适的节点
- 选择距离较近的节点
- 避免过载的节点
- 定期更换节点

### 2. 优化网络配置
```yaml
# 增加并发连接
tcp-concurrent: true

# 调整 UDP 超时
udp-timeout: 300

# 启用 TUN 模式
tun:
  enable: true
  stack: system
```

### 3. 调整 DNS
```
使用快速 DNS：
- 8.8.8.8 (Google)
- 1.1.1.1 (Cloudflare)
- 119.29.29.29 (Tencent)
```

### 4. 使用分流规则
```
只代理必要的流量：
- 国内网站直连
- 国外网站代理
- 减少不必要的代理
```

## 常见问题

### Q: 为什么同一个节点速度变化很大？
A: 
- 网络波动
- 服务器负载变化
- 时间段不同
- 测速网站差异

### Q: 如何获得最准确的测速结果？
A:
1. 多次测试取平均值
2. 在不同时间段测试
3. 使用多个测速工具
4. 排除干扰因素

### Q: 节点速度慢怎么办？
A:
1. 尝试其他节点
2. 检查网络连接
3. 重启应用
4. 更新节点信息
5. 考虑升级到付费服务
